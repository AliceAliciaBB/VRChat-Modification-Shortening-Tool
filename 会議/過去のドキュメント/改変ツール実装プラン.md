# VRChat改変手順 自動化ツール 実装プラン

## Context

`いつもの改変手順.md` には、Modular Avatar (MA) を使ったVRChatアバター改変の定型作業がまとめられている。具体的には「メニュー用の空オブジェクト階層を作る」「衣装をSetup Outfitしてトグル化する」「距離フェードをマテリアルに一括適用する」「髪型同士を排他切り替えにする」といった、毎回手作業でやっている繰り返し作業が中心。これをUnityのEditorWindow(独自ツールウィンドウ)に集約し、ボタン操作で済ませられるようにする。

このリポジトリは現在 `README.md` と手順書のみで、Assetsフォルダ等のUnityプロジェクト構造は存在しない。ユーザーの回答により、このツールは **UPMパッケージとして作成**し、別の既存VRChatアバター用Unityプロジェクト(MA・VRCSDKはVCC導入済み)に追加して使う前提とする。

事前調査で、Modular Avatar (bdunderscore/modular-avatar) の実装を確認済み:
- 衣装のSetup Outfitは `nadena.dev.modular_avatar.core.editor.SetupOutfit.SetupOutfitUI(GameObject outfitRoot)` という `[PublicAPI]` 公開メソッドが存在し、これを直接呼べる(GitHub Issue #907で公式に外部API利用が確認されている)。
- 「Create Toggle for Selection」の実体ロジックは `internal` クラスのため直接呼べないが、`ModularAvatarObjectToggle` / `ModularAvatarMenuItem` / `ModularAvatarMenuInstaller` はすべて public なRuntimeコンポーネントなので、同等の処理を自前で再実装できる(実装パターンはMA本体の `ToggleCreatorShortcut.cs` を参考にする)。
- 「インストール先をM_<name>にする」は `ModularAvatarMenuInstallTarget`(public, `installer`フィールドを持つ)を使い、M_<name>の子に新規オブジェクトを作って`installer`を紐付けることで実現できる(MA本体の `MenuInstallerEditor.OpenSelectMenu` の `MenuNodesUnder` 分岐と同じ手法)。
- 髪型の排他切り替えは、VRChat Expression Menuのネイティブな仕様(複数のToggleコントロールが同じint/float parameterを共有し、それぞれ異なる非ゼロ`value`を持つと排他的になる)を使う。MAの `VRCExpressionsMenu.Control` (Controlプロパティ経由)で `parameter.name` と `value` を揃えるだけでよく、Animator側の特別な仕組みは不要。

「一括距離フェード」は標準MA機能ではなく、ユーザー提供のlilToon用エディタスクリプト(`_UseDistanceFade` / `_DistanceFadeColor` / `_DistanceFade` プロパティを操作)がベースになっている。これをツールに統合する。

## パッケージ構成 (UPMパッケージ、リポジトリ直下に作成)

```
package.json
Editor/
    Vrcmst.asmdef
    MainWindow.cs            … EditorWindowエントリポイント、タブ切り替えのみ
    MenuSetupSection.cs       … M_menuObj作成 / カテゴリ(O_<name>, M_<name>)作成
    CostumeSection.cs         … 衣装フロー: SetupOutfit→選択→トグル作成→インストール先設定
    HairstyleSection.cs       … 髪型フロー: トグル検出→排他グループ化
    DistanceFadeSection.cs    … lilToon距離フェード一括適用(ユーザー提供スクリプトを移植)
    ModularAvatarOps.cs       … MA Runtime/Editor APIを叩く共通処理(トグル生成・インストール先設定など)
```

- パッケージID: `com.aliciabb.vrc-mod-shortcut`、displayName「VRChat Modification Shortening Tool」、version `0.1.0`。VCCから直接追加できるよう `vpmDependencies` に `nadena.dev.modular-avatar` と `com.vrchat.avatars` を記載する(plain UPMとしても`package.json`だけで動作する)。
- `Vrcmst.asmdef` は `nadena.dev.modular-avatar.core` / `nadena.dev.modular-avatar.core.editor` / `VRC.SDK3A` を参照(名前参照。Unity側でGUID解決できない場合はInspectorで手動リンクが必要になる旨をコード冒頭にコメントで残す)。

## 機能ごとの実装内容

### 1. メニュー初期セットアップ
- アバター(`VRCAvatarDescriptor`)をフィールドで指定(選択中のオブジェクトから自動検出するフォールバック付き)。
- 「M_menuObj を用意」ボタン: 既に同名の子があれば再利用、なければ `CreateEmpty` して配置。

### 2. カテゴリ作成 (O_<name> / M_<name>)
- テキストフィールドにカテゴリ名を入力(例: 衣装、アクセサリ)。
- ボタン押下で:
  - アバター直下に `O_<name>` が無ければ作成。
  - `M_menuObj` 直下に `M_<name>` が無ければ作成し、`ModularAvatarMenuItem`(Control.type=SubMenu, MenuSource=Children, label=name)と `ModularAvatarMenuInstaller`(installTargetMenu=null → ルートメニューにインストール)を追加。

### 3. アイテム追加 (プレハブ→O_<name>)
- 既存の `O_<name>` をドロップダウンで選択、プレハブ参照フィールド、メニュー作成タイプ(衣装/髪型/その他)のラジオ選択。
- 「追加」ボタンで:
  1. プレハブを `O_<name>` の子としてインスタンス化。
  2. 距離フェード一括適用(§5)をそのオブジェクト以下に実行。
  3. タイプ別分岐:
     - **衣装**: `SetupOutfit.SetupOutfitUI(instance)` を呼ぶ → 直下の子から `ModularAvatarMergeArmature` が付いた(またはGameObject名が"Armature"の)オブジェクトを除外して選択 → `ModularAvatarOps.CreateTogglesForSelection(...)` で複数なら「`<name> Toggles`」サブメニューオブジェクト+トグル群、単一なら単独トグル+インストーラーを作成(MA本体のロジックを再実装) → 生成されたインストーラーに対し `ModularAvatarMenuInstallTarget` を`M_<name>`の子に作成して紐付け。
     - **髪型**: インスタンス全体に対し単一トグルを作成し、同様に `M_<name>` へインストール先を設定。後で§6の排他グループ化の対象にする。
     - **その他**: 追加のみ(手順書通り「無し」)。
  - 翻訳ステップは手順書内でも「検討のみで実装は後回し」と明記されているため、本ツールでは対象外(TODOコメントのみ残す)。

### 4. 髪型トグルの排他グループ化
- 対象スコープ(O_<name>カテゴリ、またはアバター全体)から `ModularAvatarMenuItem`(Toggle) + `ModularAvatarObjectToggle` を持つオブジェクトを列挙。タグが `EditorOnly` のものは除外して表示。
- チェックボックス付きリストで対象を選び、「排他グループとして設定」ボタンで:
  - 共有パラメータ名(デフォルトはカテゴリ名)を決定。
  - 各対象の `Control.parameter.name` を共有名に、`Control.value` を1始まりの連番に設定し、`automaticValue=false` にする。

### 5. 距離フェード一括適用
- ユーザー提供スクリプトのロジック(`liltoonBatchDistanceFeat`)をそのまま移植: フェードカラー/開始距離/終了距離/強度をフィールドで持ち、選択オブジェクト以下のRendererを再帰走査して `liltoon` シェーダーのマテリアルにのみ `_UseDistanceFade` / `_DistanceFadeColor` / `_DistanceFade` を設定。
- スタンドアロンの操作(現在の選択に対して実行するボタン)としても提供し、§3の自動フローからも同じメソッドを呼ぶ。

### 手順書1行目について(参考情報、ツール化はしない)
「プレハブの上書き保存が可能か」は本ツールの自動化対象ではなく単純な質問。回答としては、アバタールートがPrefabインスタンスであれば、選択して「Overrides > Apply All」(右クリック > Prefab > Apply All)で上書き保存可能、または該当プレハブをPrefabモードで開いて直接編集・保存も可能。プラン実行後にチャットで一言触れる。

## 既知の制約・確認事項
- このリポジトリ単体ではUnity Editorが存在しないため、コンパイル確認はできない。コード作成後、ユーザーの実Unityプロジェクト(MA+VRCSDK導入済み)に取り込んでコンパイル・動作確認してもらう必要がある。
- 排他グループ化(§4)はMAのドキュメント化されていない挙動(共有パラメータ+複数value)に依存するため、実際にビルドしてVRChat上で動作確認することを推奨する旨をコード内コメント/チャットで明記する。
- asmdefの参照はアセンブリ名解決に依存するため、ユーザー環境によってはInspectorでの手動リンクが必要になる場合がある。

## 検証方法
- 実Unity環境がないため自動テストは行わない。代わりに各C#ファイルが構文的に妥当か確認(目視+API名の再確認)し、実装後にユーザーへ「実プロジェクトに配置して動作確認してください」と伝える。
